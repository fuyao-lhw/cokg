- 协同办公知识图谱后端系统开发文档

  ## 1. 项目概述

  ### 1.1 项目简介

  本项目是一个**协同办公知识图谱后端系统**，基于 Spring Boot 框架构建，提供知识图谱的管理、查询、关系维护以及 AI 助手等核心功能。

  ### 1.2 技术栈

  | 分类       | 技术         | 版本      |
  | :--------- | :----------- | :-------- |
  | 语言       | Java         | 21        |
  | 框架       | Spring Boot  | 3.2.x     |
  | 数据库 ORM | MyBatis Plus | 3.5.x     |
  | 认证       | JWT          | Auth0 JWT |
  | 缓存       | Redis        | -         |
  | JSON处理   | Jackson      | -         |

  ## 3. 认证与权限

  ### 3.1 JWT认证机制

  #### 3.1.1 Token结构

  **Payload包含信息**：
  - `iss`：签发者
  - `sub`：用户名称
  - `code`：用户编码
  - `role`：用户角色
  - `exp`：过期时间（默认2天）

  #### 3.1.2 Token验证流程

  #### 3.1.3 拦截器配置

  **需要拦截的路径**：`/**`（所有请求）

  **无需拦截的路径**：
  - `/user/login` - 登录接口
  - `/user/register` - 注册接口
  - `/user/check_auth` - 权限校验接口
  - `/graph/**` - 图数据接口

  ---

  ### 3.2 响应码枚举

  **ResultCodeEnum**

  | 响应码 | 消息         | 说明          |
  | :----- | :----------- | :------------ |
  | 200    | 操作成功     | 成功操作      |
  | 100    | 操作失败     | 通用失败      |
  | 101    | 空字符串     | Token出现问题 |
  | 401    | 操作没有权限 | 未授权        |

  ## 4. 数据访问层

  ### 4.1 Mapper接口结构

  所有Mapper接口继承MyBatis Plus的`BaseMapper<T>`，提供基础CRUD操作。

  **KnowledgeMetaMapper**
  - `selectList(Wrapper)` - 查询图谱列表
  - `selectById(Long)` - 根据ID查询图谱
  - `insert(KnowledgeMetaEntity)` - 插入图谱
  - `updateById(KnowledgeMetaEntity)` - 更新图谱
  - `deleteById(Long)` - 删除图谱（逻辑删除）

  **KnowledgeNodeMapper**
  - `selectByKgMetaId(Long)` - 根据图谱ID查询节点列表
  - `batchSelectByIds(List<Long>)` - 批量查询节点

  **KnowledgeRelationMapper**
  - `selectByKgMetaId(Long)` - 根据图谱ID查询关系列表
  - `selectByNodeId(Long)` - 根据节点ID查询相关关系

  ## 10. 代码规范

  ### 10.1 命名规范

  - **类名**：大驼峰命名（PascalCase），如 `UserController`
  - **方法名**：小驼峰命名（camelCase），如 `getUserList`
  - **变量名**：小驼峰命名，如 `userId`
  - **常量名**：全大写，下划线分隔，如 `EXPIRATION_TIME`

  ### 10.2 注释规范

  - 所有类、接口、方法必须添加Javadoc注释
  - 复杂业务逻辑需添加详细注释
  - 注释语言使用中文

  ### 10.3 异常处理

  - 使用统一异常处理机制
  - 避免直接抛出底层异常
  - 异常信息需清晰、友好
